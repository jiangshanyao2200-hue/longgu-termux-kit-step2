# PTY 后台 + 发送队列（AItermux / 萤）计划与对账

## 目标
- **后台长任务**：交互式 PTY（`bash interactive:true`）启动后立刻返回，可继续聊天；终端完成后再异步回传结果。
- **可继续聊天**：PTY 运行期间，输入框用于和 AI 聊天；PTY 视图有独立光标/输入。
- **定时审计**：PTY 运行中每 5 分钟注入一次审计提示，让模型按需读取 `status/log` 并简短回复。
- **强发送队列**：API 活跃（Generating/ExecutingTool）时 `Enter` 只换行，但 `Alt+Enter` 可把当前输入“强发送入队”（最低优先级），空闲后自动出队发送。
- **隔离与链路清晰**：高耦合的“系统自动注入提示”（例如 PTY audit）保持内置，降低误改/错配；任何大输出都落盘并返回 head/tail + 文件元信息，方便按需再读。

## PTY 行为（已实现）
- `bash interactive:true` 会启动 PTY 后台 job，并快速返回 `job_id/log/status/cwd/cmd` 和初步输出预览。
- PTY 结束后会生成 `PTY DONE`（批量）工具消息：多 tab 并发结束时先缓存进度并用系统提示显示 `PTY x/y DONE（已缓存）`，待同一批全部结束后再汇总为一条 `PTY DONE (BATCH)` tool result（附 `saved/status/size/lines` + tail 预览；超阈值落盘）。
- **多 PTY（tab）**：允许同时存在多个后台终端任务；每次启动新的 `bash interactive:true` 会追加一个 tab。
- 光标与尺寸：键盘弹出/收回导致尺寸变化时，PTY 会立即 resize；并严格对齐“渲染行数==rows”，减少光标错位/漂移。
- 键位（终端焦点时）尽量模拟真实终端：大多数按键直接转发给 PTY；但为适配多 PTY，需要占用少量切换键位。
  - `Home`：隐藏/显示 Terminal 视图（后台继续运行）
  - `Esc`：发送给终端内程序（vim/less/cancel 等）
  - `Esc×2`（350ms 内）：结束**当前**终端任务（终止当前 tab，对应 tab 立即移除；若这是最后一个 tab，则终端视图消失）
  - `PgUp/PgDn`：在终端视图内快速切换不同 tab（焦点切换；只占用无修饰键）
  - `Alt+↑`：复制当前终端快照到聊天输入框（由用户决定是否发送给 AI；若 PTY 结束/被双击 Esc 终止且未发送，会自动清理）

## 定时审计（已实现）
- 每 5 分钟触发一次（空闲门控：不与用户请求/工具并发）。
- 注入模板：内置（`DEFAULT_PTY_AUDIT_PROMPT`），不再外置到 `prompt/`。
- 内容会指明 `status/log` 路径，模型用 `read_file` 查看并简短回复即可。

## 发送队列（已实现）
### 语义
- 队列分两级：`High` 与 `Low`（**Low 是最低优先级**）。
- 当前只在 API 活跃时启用“强发送入队”（进 `Low`）；空闲后自动出队（High 优先，再 Low）。
- 优先级：`PTY DONE (BATCH)` 回执 > 发送队列 > 心跳/审计（避免队列堆积导致 PTY 回执延后）。
- 自动出队节流：每次出队后至少间隔约 `450ms` 才会继续出队，避免“刚空闲就连发多条”让人类/模型都来不及反应。

### 键位
- **API 活跃时**（Generating/ExecutingTool）：
  - `Enter`：仅换行
  - `Alt+Enter`：把当前输入入队（Low），输入框清空；状态栏显示待发数量
- **队列 UI**：
  - `Alt+↑`：打开队列列表（↑↓ 选择，Enter 进入编辑，Home 退出）
  - 编辑态：
    - `Alt+↓`：保存编辑（不改变队列顺序）
    - `Alt+Enter`：保存并把该条置底（Low 尾部）

### 状态栏
- 当存在待发：在提示行显示 ` / N 条待发`
- 队列列表/编辑态会显示对应操作提示。

## 大输出策略（已实现）
- 任一工具输出超阈值导致落盘时，优先返回：
  - `saved` 文件路径 + `dir`
  - `size/lines` 元信息
  - `head/tail` 预览（避免只回 `[saved:...]` 导致看不到关键片段）
