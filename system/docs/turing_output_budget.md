# 输出预算与落盘（图灵化约定）

目标：避免一次工具调用返回过长内容导致上下文爆炸；当内容过长时，**优先落盘**并回传“指针 + 预览 + 元信息”。

## 1) 统一原则

- 工具返回给 UI/智能体的文本必须遵守“总字符/总行数预算”。
- 超出预算时：
  - 返回 **预览**（头/尾或关键片段）并明确标注已截断。
  - 将完整内容导出到 `AItermux/system/log/*`（按工具分类建子目录）。
  - 在 `log_lines` 中提供 `saved:` 指针，以及 `size/lines` 等元信息。

## 2) 对智能体的使用建议

- 需要细节时，优先根据 `saved:` 或 `log:` 再次 `read_file`（或对应工具的读取接口），而不是要求工具一次性吐出全量。
- 搜索类工具返回大量命中时，应缩小范围（路径/关键词/文件类型/深度）或改为“先列表、后精读”。

## 3) 对工具作者的实现建议

- 为每个工具定义：
  - `export_dir`（落盘目录）
  - `preview_budget`（预览预算）
  - `hard_budget`（硬预算，强截断）
- 任何落盘都应返回：
  - `saved:<path>`
  - `meta:size_bytes=<n> lines=<n>`
